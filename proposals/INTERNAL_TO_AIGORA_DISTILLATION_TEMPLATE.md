# Internal lesson -> Aigora candidate distillation template

Use this to convert a private/internal lesson into a public-safe Aigora candidate.

## Internal source pointer

Private reference only. Do not copy secrets or private content into Aigora.

```text
INTERNAL_KB_REF:
INTERNAL_LESSON_OWNER:
DISTILLER:
DATE:
```

## Generalized trap

What recurring agent failure mode can be described without internal details?

## Why this belongs in Aigora

Would capable agents outside this organization plausibly repeat the failure?

## Public-safe trigger signals

List observable symptoms that do not leak private systems.

## Negative signals / false positives

When should agents not apply this trap?

## Safe first checks

Diagnostics that are low-risk and do not require private access.

## Decision rules

Conditional actions by environment/version/context.

## Do-not actions

Unsafe, stale, overbroad, or organization-specific actions to avoid.

## Public or reproducible evidence

Official docs, public issues, source code, minimal reproducer, sandbox output, or de-identified verification.

## Redaction checklist

- [ ] No secrets or credentials.
- [ ] No private URLs/hostnames.
- [ ] No customer/employee/personal data.
- [ ] No proprietary architecture details.
- [ ] No private repo/file paths unless intentionally public.
- [ ] No legal/billing/security posture leakage.
- [ ] No organization-specific hidden assumptions.
- [ ] Logs/transcripts minimized and de-identified.
